# CortexAI — Development Walkthrough

CortexAI has been successfully developed as a modular, generic AI/ML engineering platform. It's ready to be used as a portfolio project!

## Architecture & Tech Stack

CortexAI is built using modern, production-grade tools, maximizing performance while keeping costs at zero:

*   **Backend:** FastAPI (Python), SQLAlchemy (Async SQLite)
*   **AI/ML Core:** LangChain, LangGraph
*   **Vector Database:** ChromaDB
*   **LLM Provider:** Groq (using `llama-3.3-70b-versatile` on their free tier)
*   **Embeddings:** `sentence-transformers/all-MiniLM-L6-v2` (runs locally, 100% free)
*   **Frontend:** React, Vite, Custom Premium CSS Design System

## Key Features Developed

### 1. 🔄 Multi-Agent System (LangGraph)
The core of the system is an orchestration layer built with LangGraph. When a user submits a query, a **Router Agent** analyzes the intent and dynamically routes the request to specialized agents:
*   **Search Agent:** Focuses on extracting facts and answering questions from the knowledge base.
*   **Chat Agent:** Maintains conversation history and handles conversational follow-ups.
*   **Recommend Agent:** Analyzes user profiles against product metadata.

### 2. 🔍 RAG Pipeline & Semantic Search
*   Documents are automatically chunked and embedded using local HuggingFace models.
*   ChromaDB provides fast vector similarity search.
*   The LLM generates summaries and direct answers based **only** on retrieved context, reducing hallucinations.

### 3. 💬 Real-Time Streaming Chat
*   The frontend connects to the FastAPI backend via **WebSockets**.
*   The Chat agent streams tokens individually as they are generated by Groq, providing a highly responsive "typing" effect.

### 4. 🎯 Blended Recommendations
The Recommendation Engine goes beyond simple vector search. It uses a **Blended Scoring System**:
*   `60%` Semantic Similarity (from embeddings)
*   `20%` Category Match
*   `20%` Priority/Tag Alignment
The LLM then generates personalized insights explaining *why* each item was recommended.

### 5. 📊 Agent Monitoring Dashboard
A dedicated dashboard provides observability into the AI system:
*   Real-time system health (DB, Chroma, Groq).
*   Agent execution logs showing exact paths taken through the LangGraph nodes.
*   Query intent distribution and latency metrics.

## 🚀 How to Test Locally

Both the backend and frontend development servers are currently running!

1.  Open the frontend application in your browser: [http://localhost:5173](http://localhost:5173)
2.  Navigate through the different features using the top navigation bar.
3.  **Try the Chat:** Ask questions like *"What AI tools do you have for teachers?"*
4.  **Try Recommendations:** Fill out the wizard to see the blended scoring in action.
5.  **Check the Monitor:** Go to the Monitor page to see how your queries were routed by LangGraph.
